Functions

★ Functions:

Functions are reusable blocks of code that can be called multiple times. JavaScript allows you to define functions with parameters and return values.

  • ➤    A function is a block of code that performs a specific task and can be reused throughout your code.
  • ➤    In JavaScript, functions are first-class citizens, meaning they can be passed as arguments to other functions, returned from functions, and stored in variables.
  • ➤    Functions can be defined using the function keyword, as well as using arrow functions, which were introduced in ECMAScript 6.
  • ➤    A function can accept parameters, which are variables that are passed into the function when it is called. These parameters are used within the function to perform its task.
  • ➤    A function can also return a value using the return keyword. This value can then be used by the calling code.
  • ➤    Functions can have local variables, which are variables that are defined within the function and are only accessible within that function.
  • ➤    Functions can also have global variables, which are variables that are defined outside of any function and are accessible throughout your code.
  • ➤    JavaScript has several built-in functions, such as console.log() and alert(), which perform common tasks.
  • ➤    Functions can be used to create objects, which are collections of properties and methods that can be used to perform specific tasks.
  • ➤    JavaScript also has anonymous functions, which are functions that do not have a name and are defined inline. These functions are often used as callbacks, which are functions that are passed as arguments to other functions and are called when a specific event occurs.
  • ➤    A function in JavaScript is a block of code designed to perform a particular task.
  • ➤    Functions can be defined using the function keyword followed by the function name, parameter list (enclosed in parentheses), and the function body (enclosed in curly braces).
  • ➤    Functions can also be defined using arrow notation (=>) for shorter and more concise code.
  • ➤    Functions can take parameters (values) as inputs and can return a value as output using the return keyword.
  • ➤    Functions can be called by their name followed by parentheses containing any arguments passed to them.
  • ➤    Functions can be assigned to variables or passed as arguments to other functions.
  • ➤    Functions can be declared and called later in the code using function expressions.
  • ➤    Functions can be defined inside other functions (nested functions) to limit their scope and increase code modularity.
  • ➤    Functions have their own scope and can access variables defined in their outer (enclosing) scope. This is known as closure.
  • ➤    Functions can also be used as constructors to create new objects using the new keyword.
  • ➤    Function declarations are hoisted (moved to the top of the code) during the compilation phase of JavaScript, which allows them to be called before they are defined in the code.
  • ➤    JavaScript has several built-in functions that can be used for common tasks, such as console.log() for printing to the console and Math.random() for generating random numbers.

★ Control Flow:

  • ➤    Functions are one of the key building blocks of JavaScript. They are used to group a set of related statements together to perform a specific task. There are several types of functions in JavaScript, including:

    • 1․ Named Functions:

  • ➤    These are functions that are declared with a name and can be called using that name. They can be defined using the function keyword followed by the function name and parameter list.

    • 2․ Anonymous Functions:

  • ➤    These are functions that are not named and are often used as arguments to other functions or as the return value of a function. They can be defined using the function keyword without a name.

    • 3․ Arrow Functions:

  • ➤    These are a more concise way of writing functions and were introduced in ES6. They use the => syntax instead of the function keyword and can be used to write both named and anonymous functions.

    • 4․ Higher-Order Functions:

  • ➤    These are functions that take other functions as arguments or return functions as their return value. They are often used for tasks such as filtering or mapping arrays.

    • 5․ Constructor Functions:

  • ➤    These are functions that are used to create new objects. They are typically defined using the function keyword and the new operator is used to create new instances of the object.

    • 6․ Callback Functions:

  • ➤    These are functions that are passed as arguments to another function and are called when a certain event or condition occurs. They are commonly used in asynchronous programming.

    • 7․ Immediately Invoked Function Expressions (IIFEs):

  • ➤    These are functions that are defined and immediately called without being assigned to a variable. They are often used to create a local scope for variables and functions to avoid polluting the global scope.
